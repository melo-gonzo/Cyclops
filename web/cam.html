Motion detection

Every 0.5s each block's average brightness is compared against the previous check.
grid — how finely the frame is split into blocks (across×down shown at right). Finer pinpoints motion better but each block averages fewer pixels, so it reads noisier.
blocks to trigger — how many blocks must change, on 2 checks in a row, to record a clip. Lower = more sensitive.
pixel diff — brightness change (0–255) a block needs before it counts as changed. Lower = more sensitive; keep it above "avg diff" or noise will count.
level — how many blocks changed in the last check; a clip triggers when level ≥ blocks-to-trigger twice in a row.
luma — average frame brightness (0–255). In the dark the sensor slows down and noise rises.
avg diff — frame-wide average per-pixel change between checks: the noise floor of the scene. A static scene reads 1–3.
Turn on the "motion" overlay above to tune visually: red = changing now, blue = masked out (never counts). The mask clears if the resolution or grid changes.
